# MinerReview 项目流程图

## 整体架构

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                           ReviewPipeline                                     │
│                        (主流程控制器)                                         │
└─────────────────────────────────────────────────────────────────────────────┘
                                      │
                                      ▼
┌─────────────────────────────────────────────────────────────────────────────┐
│                              1. Retriever                                    │
│                           (检索相关论文)                                      │
│  ┌─────────────┐    ┌─────────────┐    ┌─────────────┐                      │
│  │  Milvus     │    │   FAISS     │    │  Embedding  │                      │
│  │ Vector DB   │    │   Index     │    │  (BGE/VLLM) │                      │
│  └─────────────┘    └─────────────┘    └─────────────┘                      │
│                                                                              │
│  输入: Target Paper (title, abstract, fulltext)                             │
│  输出: related_papers, related_reviews, unrelated_papers                    │
└─────────────────────────────────────────────────────────────────────────────┘
                                      │
                                      ▼
┌─────────────────────────────────────────────────────────────────────────────┐
│                           2. CriteriaMiner                                   │
│                          (挖掘评估标准)                                       │
│                                                                              │
│  ┌────────────────────────┐    ┌────────────────────────┐                   │
│  │  Content Criteria      │    │  Policy Criteria       │                   │
│  │  (从相关论文/评论挖掘)   │    │  (从 venue policy 挖掘) │                   │
│  └────────────────────────┘    └────────────────────────┘                   │
│                                                                              │
│  使用 LLM (qwen-plus) 生成结构化评估标准                                      │
│  输出: Criterion[] {text, theme, kind, source_ids}                          │
└─────────────────────────────────────────────────────────────────────────────┘
                                      │
                                      ▼
┌─────────────────────────────────────────────────────────────────────────────┐
│                         3. CriteriaDistiller                                 │
│                          (去重和筛选)                                        │
│                                                                              │
│  - 基于语义相似度去重 (dedup_threshold=0.86)                                 │
│  - 按主题均衡筛选 (max_total=12, max_per_theme=4)                           │
│  - 使用 Embedding 计算相似度                                                 │
└─────────────────────────────────────────────────────────────────────────────┘
                                      │
                                      ▼
┌─────────────────────────────────────────────────────────────────────────────┐
│                         4. CriteriaRewriter                                  │
│                         (重写评估标准)                                        │
│                                                                              │
│  使用 LLM 将通用标准改写为针对目标论文的具体问题                               │
│  例: "实验是否充分" → "Figure 3 的消融实验是否验证了模块有效性?"              │
└─────────────────────────────────────────────────────────────────────────────┘
                                      │
                                      ▼
┌─────────────────────────────────────────────────────────────────────────────┐
│                        5. ThemeAgents (并行)                                 │
│                          (主题评审)                                          │
│                                                                              │
│  ┌──────────┐ ┌──────────┐ ┌──────────┐ ┌──────────┐ ┌──────────┐          │
│  │ Quality  │ │ Clarity  │ │Originality│ │Experiments│ │  ...     │          │
│  │  Agent   │ │  Agent   │ │  Agent   │ │  Agent   │ │  Agent   │          │
│  └──────────┘ └──────────┘ └──────────┘ └──────────┘ └──────────┘          │
│       │            │            │            │            │                 │
│       ▼            ▼            ▼            ▼            ▼                 │
│  strengths,   strengths,   strengths,   strengths,   strengths,            │
│  weaknesses   weaknesses   weaknesses   weaknesses   weaknesses            │
│                                                                              │
│  每个 Agent 使用 LLM 独立评估其负责的主题                                     │
│  输出: ThemeOutput {theme, strengths, weaknesses, severity_tags}            │
└─────────────────────────────────────────────────────────────────────────────┘
                                      │
                                      ▼
┌─────────────────────────────────────────────────────────────────────────────┐
│                          6. ArbiterAgent                                     │
│                           (仲裁者)                                           │
│                                                                              │
│  汇总所有 ThemeOutput:                                                       │
│  - 合并 strengths 和 weaknesses                                              │
│  - 根据 ICLR 评分标准 (1-10) 给出评分                                        │
│  - 根据 policy criteria 检查格式要求                                         │
│  - 给出 accept/reject/borderline 决策                                       │
│                                                                              │
│  输出: ArbiterOutput {strengths, weaknesses, raw_rating, decision, ...}     │
└─────────────────────────────────────────────────────────────────────────────┘
                                      │
                                      ▼
┌─────────────────────────────────────────────────────────────────────────────┐
│                       7. SimilarReviewScoring                                │
│                      (相似评论评分校准)                                       │
│                                                                              │
│  检索相似历史评论，参考其评分调整当前评分:                                     │
│  - 检索 top_k=8 篇相似评论                                                   │
│  - LLM 根据历史评分调整当前评分                                               │
└─────────────────────────────────────────────────────────────────────────────┘
                                      │
                                      ▼
┌─────────────────────────────────────────────────────────────────────────────┐
│                          8. Calibrator                                       │
│                          (评分校准)                                          │
│                                                                              │
│  使用 Isotonic Regression 校准评分:                                         │
│  - 基于历史 accept/reject 分布                                               │
│  - 输出 acceptance_likelihood (0-1)                                         │
└─────────────────────────────────────────────────────────────────────────────┘
                                      │
                                      ▼
┌─────────────────────────────────────────────────────────────────────────────┐
│                        9. MemoryUpdate                                       │
│                        (记忆更新)                                            │
│                                                                              │
│  更新 policy criteria 的使用统计:                                            │
│  - 记录评分结果                                                              │
│  - 更新 stable/borderline 状态                                               │
└─────────────────────────────────────────────────────────────────────────────┘
```

                              │

## LLM 调用统计 (每篇论文)

| 步骤 | 调用次数 | 说明 |
|------|---------|------|
| CriteriaMiner.content | 1 | 挖掘内容评估标准 |
| CriteriaMiner.policy | 1 | 挖掘政策评估标准 |
| CriteriaRewriter | 1 | 重写评估标准 |
| ThemeAgents | 4-6 | 并行执行 |
| ArbiterAgent | 1 | 汇总仲裁 |
| SimilarReviewScoring | 1 | 相似评论校准 |
| **总计** | **~9-11** | 并行后有效 ~6 次 |
